# Stage 5.4 — Phase 2 (Session Details Redesign) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the Session Details editor to an explicit Save/Discard model, replace the contradictory speaker UI with identity-first speaker slots (named + unnamed) durably linked to diarised voices via `ClusterKey`, replace the matter checkbox wall with a searchable multi-select picker (rolled out to three surfaces), and add untag-from-matter — spec `docs/plans/2026-07-05-stage-5.4-uiux-overhaul-design.md` §5 (Phase 2), locked decisions §3/§10.

**Architecture:** WPF desktop app, strict MVVM with WPF-free ViewModels (dialogs/confirms via injected seams or humble code-behind); CommunityToolkit.Mvvm source generators; all disk writes via `MaintenanceService` under its per-session/index gates; evidentiary firewall (transcript.jsonl/audio never mutated; `Edited`/`LastEditedAtUtc`/`Summary*` flow through `BuildMeta` untouched). Builds on Phase 1 (landed @ c071542): the `Saved` event/grid refresh, `SectionGrouper`, and the playback stack are consumed, not modified.

**Tech Stack:** .NET 10 / C# / WPF / WPF-UI (`ui:`) / CommunityToolkit.Mvvm / xUnit. Build & test with `dotnet build` / `dotnet test`.

## Global Constraints

Copied from spec §2 — every task's requirements implicitly include these:

- **Evidentiary firewall:** `transcript.jsonl` and audio are **never** mutated; metadata edits touch only `meta.json`; diarisation/name changes are non-destructive overlays in `speakers.json`; `Edited` / `LastEditedAtUtc` / `Summary*` are set only by `EditStore` and flow through `BuildMeta` untouched; no hide/delete/redact of content.
- **Single disk-write owner:** all persistence via `MaintenanceService` (`SaveMetaAsync` with previous-tags delta, `SaveDiarisationAsync`, `CreateMatterAsync`, `LoadSessionItemAsync`, …); ViewModels never touch stores directly; the matter `SessionCount` delta invariant must never gain a second write path.
- **WPF-free ViewModels:** no WPF types in VMs; background→UI via injected `Action<Action> _dispatch`; user confirms via injected `Func<string,bool>` seams (composition root supplies MessageBox); timers surface as `Tick()`.
- **CommunityToolkit.Mvvm:** `[ObservableProperty]` + generated `OnXxxChanged` partials; `IRelayCommand`/`IAsyncRelayCommand` with real `CanExecute` + `NotifyCanExecuteChanged`.
- **Theme:** `DynamicResource` WPF-UI brushes only; **no hardcoded ARGB**; no keyless implicit `TextBlock` style; page/window roots keep the `TextElement.Foreground` marker (all enforced by `XamlHygieneTests`).
- **No Unicode emojis** in tests or code. **Zero new build warnings.** Core.Tests has 2 pre-existing missing-fixture failures (`Der_within_baseline_plus_epsilon`, `Golden_pair_wer_stays_at_baseline`) — not regressions.
- **Humble objects** (windows, XAML, MessageBox, composition root) get no unit tests — logic lives in tested VMs; XAML tasks carry manual-verification steps.

## Locked design decisions (do not revisit during implementation)

1. **Buffer everything until Save** (§5.1): no field/collection/tag change writes disk; `SaveCommand` commits once; `DiscardCommand` reverts to `_savedMeta`; close-when-dirty prompts Save/Discard/Cancel; Save warns via the confirm seam when speaker attribution changes. Matters-tab panel stays auto-save; Split-speakers keeps its own Confirm.
2. **Identity-first slots** (§5.2): per-side slot list (Named + explicit Unnamed via `ParticipantKind`); count boxes + `CountsFollowLists` checkbox removed; persisted `LocalCount`/`RemoteCount` = slot count on commit (pipeline-facing, floor 1); lazy migration synthesizes Unnamed slots on load (never dirties, never writes until Save).
3. **Diarise gate (clean editor):** `DiariseCommand` additionally requires `!IsDirty` — Split-speakers reads counts from disk, so launching from a dirty buffer would diarise with stale counts. After a Split confirm, the guaranteed-clean editor reloads from disk (no dirty-merge machinery anywhere).
4. **ClusterKey ownership** (§5.2): Split-speakers candidates carry participant identity; on Confirm, participant-owned clusters get `ClusterKey` set and persisted with speakers.json through one gated `SaveDiarisationAsync` write; re-diarise preserves ownership through `SpeakersMerge`'s collision-avoidance remap (remap table exposed to update meta).
5. **Matter picker** (§5.3): search box + bounded results + removable chips; matches Name+Reference+Id; archived revealed by search with "(archived)" suffix; inline create (creation immediate/additive, the tag buffered); same pattern on the Sessions-page filter (single-select semantics kept) and the Matters manager list.
6. **Untag** (§5.4): confirm dialog; refused while a Session Details window is open for that session (WindowRegistry); `SaveMetaAsync` previous-tags delta; card + grid refresh after.

**`_saveChain`/`_saveSeq` disposition (justification, as required):** both are DELETED. They existed to serialize a *queue* of auto-saves and to detect "a newer save is enqueued behind this one" before lighting the toast. With explicit save there is no queue: `AsyncRelayCommand`'s default no-concurrent-execution means CanExecute is false while a save runs, so at most one save is ever in flight and completion order is trivially program order. The single residual race — a field edit landing while the disk write is awaited — is covered by a monotonic `_dirtyGen` counter: `SaveAsync` snapshots it and only clears `IsDirty` if no `MarkDirty` bumped it mid-flight (the disk holds the snapshot, the editor correctly stays dirty). `_lastSavedTags`, `_savedByRow`, and `_savedMeta` keep their exact Phase-1 roles.

    /// <summary>Explicit commit (Stage 5.4 5.1): projects the working copy through BuildMeta -
    /// Edited/LastEditedAtUtc/Summary* flow through UNTOUCHED - and writes ONCE via
    /// MaintenanceService.SaveMetaAsync with the matter-tag delta computed against
    /// _lastSavedTags at commit time. At most one save is ever in flight (AsyncRelayCommand's
    /// default disallows concurrent execution), so there is no queue to serialize; _dirtyGen
    /// covers the one residual race - an edit landing mid-write keeps the editor dirty. On
    /// failure the edits are KEPT (still dirty, retry or Discard): the old auto-save rollback
    /// would now destroy work the user explicitly asked to persist.</summary>
    private async Task SaveAsync()
    {
        if (_row is null || !IsEditable || !IsDirty) return;
        var row = _row;
        var meta = BuildMeta();
        long gen = _dirtyGen;
        string[] previous;
        lock (_lastSavedTags)
            previous = _lastSavedTags.TryGetValue(row.Id, out var p) ? p : [];
        try
        {
            await _maintenance.SaveMetaAsync(row.Id, meta, previous, CancellationToken.None)
                .ConfigureAwait(false);
            lock (_lastSavedTags) _lastSavedTags[row.Id] = meta.MatterIds.ToArray();
            _dispatch(() =>
            {
                // Remember what we wrote for THIS row object even if the user moved on, so a
                // later re-Attach re-seeds from what we saved, not the stale listing snapshot.
                _savedByRow.AddOrUpdate(row, meta);
                if (!ReferenceEquals(_row, row)) return;    // user moved on mid-save
                _savedMeta = meta;
                if (gen == _dirtyGen) IsDirty = false;      // an edit during the write stays dirty
                Saved?.Invoke(row.Id);                      // explicit-commit grid refresh hook
            });
        }
        catch (Exception ex)
        {
            _dispatch(() => _errors.Report("Saving session details", ex));
        }
    }

    /// _savedMeta (the last successful commit, or the load snapshot) and clears the dirty
    /// flag. RevertToSaved repopulates under _loading, so the reload cannot re-mark dirty.</summary>
    private void Discard()
    {
        RevertToSaved();
        IsDirty = false;

    /// <summary>Stage 5.4 5.1 close guard: a dirty editor prompts Save / Discard / Cancel
    /// (Yes/No/Cancel), fixing the "title typed then X" data-loss path. WPF cannot await inside
    /// OnClosing, so Save CANCELS this close, awaits the commit, then re-Closes with
    /// _closeConfirmed set. Mirrors MattersPage.OnDeleteMatter's MessageBox confirm pattern.</summary>
    protected override void OnClosing(CancelEventArgs e)
    {
        base.OnClosing(e);
        if (_closeConfirmed) return;
        // Force-commit a focused LostFocus-bound TextBox (the speaker-count boxes) so IsDirty
        // and the VM working copy reflect what is on screen before we decide anything.
        if (Keyboard.FocusedElement is TextBox tb)
            tb.GetBindingExpression(TextBox.TextProperty)?.UpdateSource();
        if (!_vm.IsDirty) return;

        var choice = MessageBox.Show(
            "Save changes to this session before closing?\n\nYes saves, No discards the changes, Cancel keeps editing.",
            "Unsaved changes", MessageBoxButton.YesNoCancel, MessageBoxImage.Warning,
            MessageBoxResult.Cancel);
        if (choice == MessageBoxResult.Cancel) { e.Cancel = true; return; }
        if (choice == MessageBoxResult.No)
        {
            _vm.DiscardCommand.Execute(null);   // revert, then let the close proceed
            return;
        }
        e.Cancel = true;                        // Yes: the commit is async - stop THIS close
        _ = SaveThenCloseAsync();
    }

    /// <summary>Awaits the explicit save, then closes only if it actually settled clean.
    /// Ordering is safe: SaveAsync posts its completion (IsDirty/Saved) via Dispatcher
    /// BeginInvoke BEFORE its task completes, and this await's own dispatcher continuation is
    /// queued after task completion, so IsDirty here reliably reflects the outcome. A failed
    /// save (error already reported) or a declined attribution warning leaves the editor dirty
    /// and the window OPEN. SaveAsync catches all exceptions, so the discard is safe.</summary>
    private async System.Threading.Tasks.Task SaveThenCloseAsync()
    {
        await _vm.SaveCommand.ExecuteAsync(null);
        if (_vm.IsDirty) return;                // save failed or was declined - stay open
        _closeConfirmed = true;
        Close();
    }

```csharp
using LocalScribe.Core.Audio;
using LocalScribe.Core.Model;
namespace LocalScribe.Core.Projection;

/// <summary>Resolves a segment's display name per spec section 1.3 + Stage 5.4 section 5.2:
/// participant-owned cluster -> diarised/pinned assignment name -> single-declared-participant ->
/// baseline Me/Them. Pure projection: reads meta.json/speakers.json models, never writes.</summary>
public static class NameResolver
{
    public static string Resolve(TranscriptLine segment, Speakers? speakers, SessionMeta meta)
    {
        string sourceKey = segment.Source.ToString();          // "Local" / "Remote"
        SourceKind side = segment.Source == TranscriptSource.Local ? SourceKind.Local : SourceKind.Remote;

        // 1) diarisation / pinned assignment
        if (speakers is not null
            && speakers.Assignments.TryGetValue(sourceKey, out var bySeq)
            && bySeq.TryGetValue(segment.Seq.ToString(), out string? clusterKey))
        {
            // 1a) ownership (Stage 5.4 section 5.2): a NAMED slot durably owns the detected
            // voice bound to it - its meta.json Name wins over the speakers.json overlay, so
            // renaming the slot relabels its lines WITHOUT rewriting speakers.json. An Unnamed
            // owner has an empty Name by design and falls through: the design renders unnamed
            // slots "Speaker N", which is exactly the overlay/derived tiers below.
            SessionParticipant? owner = meta.Participants.FirstOrDefault(p =>
                p.ClusterKey == clusterKey
                && p.Kind == ParticipantKind.Named
                && !string.IsNullOrEmpty(p.Name));
            if (owner is not null) return owner.Name;

            // 1b) speakers.json name overlay, else the derived per-cluster label.
            if (speakers.Names.TryGetValue(clusterKey, out string? named)) return named;
            int colon = clusterKey.IndexOf(':');
            string clusterId = colon >= 0 ? clusterKey[(colon + 1)..] : clusterKey;
            return "Speaker " + clusterId;
        }

        // 2) single declared participant on that side
        int declared = side == SourceKind.Local ? meta.LocalCount : meta.RemoteCount;
        if (declared == 1)
        {
            var only = meta.Participants.FirstOrDefault(p => p.Side == side);
            if (only is not null) return only.Name;
        }

        // 3) baseline label, else derive from source
        if (!string.IsNullOrEmpty(segment.SpeakerLabel)) return segment.SpeakerLabel;
        return side == SourceKind.Local ? "Me" : "Them";
    }
}

using LocalScribe.Core.Model;

namespace LocalScribe.Core.Diarisation;

/// <summary>The result of one <see cref="SpeakersMerge.Merge(Speakers?, DiarisationCommit, IReadOnlyCollection{string})"/>:
/// the merged overlay plus the collision-avoidance remap that was applied to the commit's FRESH
/// clusterKeys (old fresh key -> new key; empty when nothing collided). A caller that stamps
/// <see cref="SessionParticipant.ClusterKey"/> from pre-merge fresh keys MUST translate them
/// through <see cref="FreshKeyRemap"/>, or an identity could end up bound to a protected
/// (pinned/owned) key that belongs to a different voice.</summary>
public sealed record SpeakersMergeResult(
    Speakers Speakers,
    IReadOnlyDictionary<string, string> FreshKeyRemap);

/// <summary>Merges a fresh diarisation run (<see cref="DiarisationCommit"/>) into the existing
/// <see cref="Speakers"/> record without ever mutating either input. Manual pins (assignment and
/// name) survive re-diarisation verbatim; because diarisation cluster ids restart at 0 each run, a
/// fresh clusterKey can collide with a surviving pinned clusterKey - such fresh keys are remapped
/// to unused ids so a different speaker can never inherit a pinned speaker's key or name.
/// Stage 5.4 (section 5.2): participant-OWNED clusterKeys (SessionParticipant.ClusterKey) are
/// protected exactly like pinned keys - a colliding fresh key is remapped away, so a different
/// voice can never be re-bound under a key a named identity owns. Pure: no IO; meta.json is
/// neither read nor written here - owners are passed in and the remap is returned.</summary>
public static class SpeakersMerge
{
    /// <summary>Stage-5 two-arg shape, kept verbatim for existing call sites and tests:
    /// no owned keys, remap discarded. Behavior is byte-identical to before Stage 5.4.</summary>
    public static Speakers Merge(Speakers? existing, DiarisationCommit commit)
        => Merge(existing, commit, []).Speakers;

    /// <summary>Returns a new <see cref="Speakers"/> combining <paramref name="existing"/> with the
    /// re-diarised <paramref name="commit"/>, plus the fresh-key remap. Pinned seqs keep their old
    /// assignment and name; every non-pinned assignment/name for a re-diarised source is replaced
    /// by the fresh run. Fresh keys colliding with pinned OR <paramref name="ownedClusterKeys"/>
    /// are remapped to unused ids. Neither input is mutated.</summary>
    public static SpeakersMergeResult Merge(Speakers? existing, DiarisationCommit commit,
        IReadOnlyCollection<string> ownedClusterKeys)
    {
        existing ??= new Speakers();
        var reSources = commit.Sources.Select(s => s.ToString()).ToHashSet(); // "Local"/"Remote"

        // Collision-avoidance remap: fresh clusterKeys restart at 0 each run, so a fresh key can
        // equal a surviving PROTECTED key - one a pinned seq maps to, or one a participant slot
        // owns (both are a DIFFERENT speaker than the fresh voice). Remap those fresh keys to
        // unused ids per source, deterministically, before applying the commit. Keyed by full
        // "Source:id" clusterKey (source-prefixed, so unambiguous across sources); non-colliding
        // keys untouched.
        var remap = new Dictionary<string, string>(StringComparer.Ordinal);
        foreach (var source in commit.Sources)
        {
            var sourceKey = source.ToString();

            // Protected keys: clusterKeys this source's pinned seqs currently map to (preserved
            // verbatim by merge) PLUS participant-owned keys for this source (Stage 5.4).
            var protectedKeys = new HashSet<string>(StringComparer.Ordinal);
            if (existing.Pinned.TryGetValue(sourceKey, out var pinnedSeqList) &&
                existing.Assignments.TryGetValue(sourceKey, out var existingBySeq))
                foreach (var seq in pinnedSeqList)
                    if (existingBySeq.TryGetValue(seq, out var ck)) protectedKeys.Add(ck);
            foreach (var ck in ownedClusterKeys)
                if (ck.StartsWith(sourceKey + ":", StringComparison.Ordinal)) protectedKeys.Add(ck);
            if (protectedKeys.Count == 0) continue; // nothing for a fresh key to collide with

            // Fresh clusterKeys this source's commit produces: assignment values + own Names keys.
            var freshKeys = new HashSet<string>(StringComparer.Ordinal);
            if (commit.Assignments.TryGetValue(sourceKey, out var freshBySeq))
                foreach (var ck in freshBySeq.Values) freshKeys.Add(ck);
            foreach (var ck in commit.Names.Keys)
                if (ck.StartsWith(sourceKey + ":", StringComparison.Ordinal)) freshKeys.Add(ck);

            var colliding = freshKeys.Where(protectedKeys.Contains).ToList();
            if (colliding.Count == 0) continue;

            // New ids start above the max id seen among all protected+fresh keys for this source,
            // so a remapped key cannot collide with any protected id, any fresh id, or another
            // new id.
            var maxId = -1;
            foreach (var ck in protectedKeys) maxId = Math.Max(maxId, ParseId(ck));
            foreach (var ck in freshKeys) maxId = Math.Max(maxId, ParseId(ck));

            var nextId = maxId + 1;
            foreach (var oldKey in colliding.OrderBy(ParseId))
                remap[oldKey] = $"{sourceKey}:{nextId++}";
        }

        // Apply the remap to a COPY of the commit's Assignments (values) and Names (keys). The
        // passed-in commit is never mutated; non-colliding keys pass through unchanged.
        var commitAssignments = commit.Assignments;
        var commitNames = commit.Names;
        if (remap.Count > 0)
        {
            var remappedAssignments = new Dictionary<string, IReadOnlyDictionary<string, string>>();
            foreach (var (src, bySeq) in commit.Assignments)
            {
                var copy = new Dictionary<string, string>();
                foreach (var (seq, ck) in bySeq)
                    copy[seq] = remap.TryGetValue(ck, out var nk) ? nk : ck;
                remappedAssignments[src] = copy;
            }
            commitAssignments = remappedAssignments;

            var remappedNames = new Dictionary<string, string>();
            foreach (var (ck, name) in commit.Names)
                remappedNames[remap.TryGetValue(ck, out var nk) ? nk : ck] = name;
            commitNames = remappedNames;
        }

        var assignments = existing.Assignments.ToDictionary(
            kv => kv.Key, kv => new Dictionary<string, string>(kv.Value));
        var pinned = existing.Pinned.ToDictionary(kv => kv.Key, kv => new List<string>(kv.Value));
        var names = new Dictionary<string, string>(existing.Names);

        // clusterKeys still referenced by any pinned seq (across all sources) must keep their names.
        var pinnedClusterKeys = new HashSet<string>();
        foreach (var (src, seqs) in pinned)
            if (assignments.TryGetValue(src, out var bySeq))
                foreach (var seq in seqs)
                    if (bySeq.TryGetValue(seq, out var ck)) pinnedClusterKeys.Add(ck);

        foreach (var sourceKey in reSources)
        {
            var pinnedSeqs = pinned.TryGetValue(sourceKey, out var ps) ? new HashSet<string>(ps) : [];

            // Reset non-pinned assignments for this source: keep only pinned seqs...
            var kept = new Dictionary<string, string>();
            if (assignments.TryGetValue(sourceKey, out var old))
                foreach (var (seq, ck) in old)
                    if (pinnedSeqs.Contains(seq)) kept[seq] = ck;
            // ...then apply the new run (skip any seq that is pinned).
            if (commitAssignments.TryGetValue(sourceKey, out var fresh))
                foreach (var (seq, ck) in fresh)
                    if (!pinnedSeqs.Contains(seq)) kept[seq] = ck;
            assignments[sourceKey] = kept;

            // Drop non-pinned Names whose clusterKey belongs to this source. (Owned keys' stale
            // overlay entries drop too: post-reset no assignment references them, and the owning
            // identity's display name lives in meta via the NameResolver ownership tier.)
            foreach (var ck in names.Keys.ToList())
                if (ck.StartsWith(sourceKey + ":", StringComparison.Ordinal) && !pinnedClusterKeys.Contains(ck))
                    names.Remove(ck);
        }

        // Apply the run's names (defaults or user-typed), using the remapped commit. Defense in
        // depth: never overwrite a pinned clusterKey's name, and only overlay names for a
        // re-diarised source. After the remap a protected-key collision cannot occur, but this
        // guard keeps a pinned name unassailable and stops a stray non-committed-source name
        // from leaking in.
        foreach (var (ck, name) in commitNames)
            if (!pinnedClusterKeys.Contains(ck) &&
                reSources.Any(src => ck.StartsWith(src + ":", StringComparison.Ordinal)))
                names[ck] = name;

        var diarisedSources = existing.DiarisedSources
            .Concat(commit.Sources).Distinct().ToList();

        var merged = existing with
        {
            Assignments = assignments,
            Pinned = pinned,
            Names = names,
            DiarisedSources = diarisedSources,
            Method = commit.Method,
            DiarisedAtUtc = commit.DiarisedAtUtc,
        };
        return new SpeakersMergeResult(merged, remap);
    }

    // Parses the integer id from a "Source:id" clusterKey (suffix after the first ':').
    // Returns -1 for a malformed key so it never inflates the allocated-id ceiling.
    private static int ParseId(string clusterKey)
    {
        var idx = clusterKey.IndexOf(':');
        return idx >= 0 && idx + 1 < clusterKey.Length &&
               int.TryParse(clusterKey.AsSpan(idx + 1), out var id)
            ? id : -1;
    }
}

    /// merge a fresh <see cref="DiarisationCommit"/> into speakers.json (pin- AND
    /// ownership-preserving via SpeakersMerge), flip session.Diarised, then regenerate
    /// projections - all under the same per-session gate SaveMetaAsync/SetArchivedAsync use.
    /// Participant-owned clusterKeys read from meta.json are protected like pins: a colliding
    /// fresh key is remapped away so a different voice can never be re-bound under a key a
    /// named identity owns. meta.json itself is READ-ONLY here - owned keys never change in a
    /// merge, and fresh-key ClusterKey fix-ups are the Split-confirm flow's job via the
    /// RETURNED remap (old fresh key -> final key; empty when nothing collided or the session
    /// was deleted mid-run). Write order matters: speakers.json (source of truth) FIRST, then
    /// the Diarised flag, then projections - so a crash between steps never advertises a
    /// diarisation whose overlay didn't land. Never flips meta.json Edited/LastEditedAtUtc
    /// (reserved for manual corrections) and NEVER deletes/touches audio for any
    /// AudioRetention value - the retained legs are primary evidence (no SessionDeleter,
    /// no IRecycleBin, no per-source removal here, ever).</summary>
    public Task<IReadOnlyDictionary<string, string>> SaveDiarisationAsync(
        string sessionId, DiarisationCommit commit, CancellationToken ct) =>
        RunForSessionAsync<IReadOnlyDictionary<string, string>>(sessionId, async inner =>
        {
            if (!File.Exists(paths.SessionJson(sessionId)))
                return new Dictionary<string, string>();            // deleted mid-run guard

            // 1) merge into speakers.json (pin- and ownership-preserving) and save FIRST
            //    (source of truth). Owned keys come from the CURRENT meta.json under this
            //    same gate, not a caller snapshot.
            var meta = await new MetadataStore(paths.MetaJson(sessionId)).LoadAsync(inner);
            var owned = meta?.Participants
                .Where(p => !string.IsNullOrEmpty(p.ClusterKey))
                .Select(p => p.ClusterKey!)
                .ToList() ?? [];
            var store = new SpeakersStore(paths.SpeakersJson(sessionId));
            var existing = await store.LoadAsync(inner);
            var result = SpeakersMerge.Merge(existing, commit, owned);
            await store.SaveAsync(result.Speakers, inner);

            // 2) flip session.Diarised (mirror the RecoverIfNeededAsync rewrite pattern).
            var sessionStore = new SessionStore(paths.SessionJson(sessionId));
            var session = await sessionStore.ReadAsync(inner);
            if (session is not null && !session.Diarised)
                await sessionStore.SaveAsync(session with { Diarised = true }, inner);

            // 3) re-render projections with the new speaker names.
            // NOTE: NO audio deletion here for any AudioRetention value (evidentiary firewall).
            await new SessionWriter(paths, settings.Current, time).RegenerateProjectionsAsync(sessionId, inner);
            return result.FreshKeyRemap;
        }, ct);

    /// merge a fresh <see cref="DiarisationCommit"/> into speakers.json (pin-preserving via
    /// SpeakersMerge), persist participant ClusterKey ownership into meta.json, flip
    /// session.Diarised, then regenerate projections - all under the same per-session gate
    /// SaveMetaAsync/SetArchivedAsync use. Write order matters: speakers.json (source of truth)
    /// FIRST, then ownership, then the Diarised flag, then projections - so a crash between steps
    /// never advertises a diarisation whose overlay didn't land. Never flips meta.json
    /// Edited/LastEditedAtUtc (reserved for manual corrections) and NEVER deletes/touches audio
    /// for any AudioRetention value - the retained legs are primary evidence (no SessionDeleter,
    /// no IRecycleBin, no per-source removal here, ever).
    /// <paramref name="participantClusterKeys"/> maps participant Id -> the run's RAW clusterKey;
    /// SpeakersMerge's collision remap is applied here so ownership always points at the key that
    /// actually landed in speakers.json. Participants are rewritten from the CURRENT meta read
    /// under the gate (SetArchivedAsync pattern) and ONLY ClusterKey changes: re-asserted slots
    /// get their (remapped) key; a re-diarised source's un-reasserted stale ownership is cleared
    /// (cluster ids restart at 0 per run - keeping it could mislabel a different voice; pinned
    /// lines keep their labels via pin-preserved speakers.Names either way); everything else,
    /// including the other side's ownership, passes through untouched. null = legacy caller.</summary>
    public Task SaveDiarisationAsync(string sessionId, DiarisationCommit commit,
        IReadOnlyDictionary<string, string>? participantClusterKeys, CancellationToken ct) =>
        RunForSessionAsync(sessionId, async inner =>
        {
            if (!File.Exists(paths.SessionJson(sessionId))) return true;   // deleted mid-run guard

            // 1) merge into speakers.json (pin-preserving) and save FIRST (source of truth).
            //    Group B's seam surfaces the collision remap applied to the commit's fresh keys.
            var store = new SpeakersStore(paths.SpeakersJson(sessionId));
            var existing = await store.LoadAsync(inner);
            var merged = SpeakersMerge.Merge(existing, commit, out var freshKeyRemap);
            await store.SaveAsync(merged, inner);

            // 1b) participant ClusterKey ownership (Stage 5.4 C2) - see doc comment above.
            if (participantClusterKeys is not null)
            {
                var metaStore = new MetadataStore(paths.MetaJson(sessionId));
                var meta = await metaStore.LoadAsync(inner);
                if (meta is not null)
                {
                    var rePrefixes = commit.Sources.Select(s => s.ToString() + ":").ToList();
                    var updated = meta.Participants.Select(p =>
                    {
                        if (participantClusterKeys.TryGetValue(p.Id, out var chosen))
                            return p with
                            {
                                ClusterKey = freshKeyRemap.TryGetValue(chosen, out var remapped)
                                    ? remapped : chosen,
                            };
                        if (p.ClusterKey is string ck &&
                            rePrefixes.Any(prefix => ck.StartsWith(prefix, StringComparison.Ordinal)))
                            return p with { ClusterKey = null };
                        return p;
                    }).ToList();
                    if (!updated.SequenceEqual(meta.Participants))   // records: value equality
                        await metaStore.SaveAsync(meta with { Participants = updated }, inner);
                }
            }

            // 2) flip session.Diarised (mirror the RecoverIfNeededAsync rewrite pattern).
            var sessionStore = new SessionStore(paths.SessionJson(sessionId));
            var session = await sessionStore.ReadAsync(inner);
            if (session is not null && !session.Diarised)
                await sessionStore.SaveAsync(session with { Diarised = true }, inner);

            // 3) re-render projections with the new speaker names + ownership.
            // NOTE: NO audio deletion here for any AudioRetention value (evidentiary firewall).
            await new SessionWriter(paths, settings.Current, time).RegenerateProjectionsAsync(sessionId, inner);
            return true;
        }, ct);

            if (!previous.Contains(matterId, StringComparer.Ordinal))
            {
                // Already untagged: write nothing, apply NO delta - re-sync the stale sublist only.
                await RefreshAsync();
                await SelectAsync(matterId);
                return;
            }
            var updated = item.Meta with
            {
                MatterIds = previous
                    .Where(id => !string.Equals(id, matterId, StringComparison.Ordinal)).ToList(),
            };
            // SaveMetaAsync applies the tag delta against the fresh on-disk previous set,
            // so the index decrement is exactly -1 for this matter (design 5.4).
            await _maintenance.SaveMetaAsync(sessionId, updated, previous, CancellationToken.None);
            await RefreshAsync();
            await SelectAsync(matterId);
        }
        catch (Exception ex) { _reporter.Report("Untag session", ex); }
    }
```

**Design decision — refuse-on-click, not a per-row disabled button.** Justification: (a) `TaggedSessionItem` is an immutable record with no `INotifyPropertyChanged`, so a per-row `CanUntag` bool would freeze at `SelectAsync` time; (b) `WindowRegistry` has no change event, so nothing would re-evaluate a binding when a Session Details window opens/closes while the Matters page is showing; (c) the check must re-run at action time anyway (a window can open between render and click). A click-time check matches the page's existing refusal idiom (`DeleteMatterAsync`'s blocked-while-referenced `Info`).
