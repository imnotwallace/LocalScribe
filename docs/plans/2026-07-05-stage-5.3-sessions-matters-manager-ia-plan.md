# Stage 5.3 Implementation Plan — Sessions & Matters Manager IA

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Manager-IA overhaul deferred by Stage 5.2 — compact icon-rail nav, a Sessions action bar + fixed right-click menu, an honest Source column, a Matters chips column, richer matter-filter labels, relocated Split-speakers, and wider matter ids.

**Architecture:** App-mostly + one Core change. The Sessions VM gains an `id→(ref,name)` matter-catalog lookup (loaded from the cheap `MattersIndex`) that feeds both the filter labels and per-row chips. `SessionRowViewModel` gains `Source` (App + capture mode) and `MatterChips`. XAML reworks the Sessions page (columns, action bar, fixed context menu) and the nav pane mode. `MatterIdGenerator` moves to a per-day id. Split-speakers relocates from the Sessions list into the Session Details window (read view keeps it).

**Tech Stack:** .NET `net10.0-windows`, WPF, Wpf.Ui 4.0.3 (`xmlns:ui="http://schemas.lepo.co/wpfui/2022/xaml"`), CommunityToolkit.Mvvm 8.4.0 (`[ObservableProperty]`, `IRelayCommand`/`IAsyncRelayCommand`), xUnit 2.9.3.

```csharp
public static string Next(MattersIndex index, string mattersDir, DateOnly date)
{
    string prefix = string.Create(CultureInfo.InvariantCulture, $"M-{date:yyyyMMdd}-");
    int max = 0;
    foreach (var entry in index.Matters)
    {
        if (entry.Id.StartsWith(prefix, StringComparison.Ordinal)
            && int.TryParse(entry.Id.AsSpan(prefix.Length), NumberStyles.None,
                CultureInfo.InvariantCulture, out int n)
            && n > max)
        {
            max = n;
        }
    }
    for (int nnn = max + 1; ; nnn++)
    {
        string candidate = string.Create(CultureInfo.InvariantCulture, $"{prefix}{nnn:D3}");
        if (index.Matters.All(e => e.Id != candidate)
            && !Directory.Exists(Path.Combine(mattersDir, candidate)))
        {
            return candidate;
        }
    }
}
```

Update the doc-comment to say `M-{yyyyMMdd}-{NNN}`, per-DAY. Then update the caller in `MaintenanceService.CreateMatterAsync` (line ~234):
