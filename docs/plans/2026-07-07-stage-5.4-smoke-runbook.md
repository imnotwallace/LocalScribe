# Stage 5.4 smoke runbook - Session Details redesign (Phase 2) + recording lifecycle (Phase 3)

GUI-only acceptance (cannot be run headless). Run on the Win11 box against a real build and record
results inline at the bottom. The 5.1-5.4 bundle is already merged to master @ 4ebd7f0 and pushed, so
this runbook runs against master; any failure becomes a fix commit on master (or a short fixup branch).

Run: `dotnet run --project src/LocalScribe.App`

## Prerequisites

- Build is green at 4ebd7f0: `dotnet build LocalScribe.slnx` (0 warnings / 0 errors);
  `dotnet test tests/LocalScribe.App.Tests` (269/269). The 2 `LocalScribe.Core.Tests` fixture failures
  (`Der_within_baseline_plus_epsilon`, `Golden_pair_wer_stays_at_baseline`) are pre-existing env-absent
  fixtures - not regressions.
- At least one finalized session; one never-edited legacy session (for P6); one session with a named
  remote speaker (for P4/P9).
- 20+ matters, some archived, at least one with a reference (for P10-P13). A scratch storage root works.
- For P9 (re-diarise) the Stage 5 diariser prerequisite applies: `LocalScribe.Diarizer.exe` published
  beside the app (self-contained single-file, `-p:IncludeNativeLibrariesForSelfExtract=true`) - see
  `docs/plans/2026-07-04-stage-5-smoke-runbook.md`. The disabled/hint checks (P8) do NOT need it.
- For H5 (per-session override) have Remote capture = perProcess in Settings and a real target app
  (Webex/CiscoCollabHost or any audio-producing process) available.
- For H8 (empty state) point the app at a fresh/empty storage root once.

# Part A - Phase 2: Session Details redesign (explicit Save/Discard, speaker slots, matter picker, untag)

## P1. Unsaved-changes indicator + Save/Discard buttons
Steps: Open Session Details for a finalized session from the Sessions grid (repeat once via a
Matters-page jump). Type one character into the Title field without tabbing away.
Expected: "Unsaved changes" appears immediately (not waiting for focus loss); Save and Discard go from
disabled to enabled; the Sessions grid row does not change yet; meta.json on disk stays unchanged.
Record:

## P2. Ctrl+S commits without losing focus
Steps: With the Title field still focused and edited (per P1), press Ctrl+S.
Expected: "Unsaved changes" disappears, Save/Discard disable again, and the Sessions grid row updates
in place with the new title (no full list reload).
Record:

## P3. Close guard: Save / Discard / Cancel prompt
Steps: Edit Title or Description, then click the window's X. Repeat once with no edits made.
Expected: Dirty case shows a Yes/No/Cancel dialog "Save changes to this session before closing?".
Cancel keeps the window open with edits + dirty indicator intact. No closes and discards (reopening
shows the old value; nothing on disk changed). Yes saves and closes (reopening shows the new value; the
grid row refreshed). Clean case: window closes instantly, no prompt.
Record:

## P4. Attribution-change warning dialog on Save
Steps: On a session with a single named remote speaker (whole-side label), remove or rename that
speaker, then click Save.
Expected: A "Session details" dialog reading "This save changes how transcript lines are attributed
(...). The transcript itself is not modified. Save anyway?" appears. No leaves the editor dirty and
disk untouched. Yes commits and refreshes the grid row.
Record:

## P5. Discard disabled during an in-flight Save
Steps: Edit a field, click Save, and immediately try clicking Discard while the save is in progress.
Expected: Discard is disabled for the brief window the save is running; it can never show reverted
fields while a stale save is about to overwrite them.
Record:

## P6. Identity-first "Speaker 1" placeholder on open (DESIGN-INTENDED)
Steps: Open Session Details for a never-edited session, and separately for a legacy system-mix session
(e.g. remoteCount 3, one named remote participant).
Expected: Every side shows at least one slot; unnamed slots render as "Speaker 1", "Speaker 2", etc.
This is DESIGN-INTENDED (declared counts == slot counts) - not a bug. Legacy example: Remote column
shows the named person plus synthesized "Speaker N" rows; opening does NOT light the Unsaved-changes
indicator; closing without saving leaves meta.json untouched.
Record:

## P7. Add-unnamed / renumber / remove speaker slots
Steps: Click the add-unnamed-speaker button on the Local column, then on the Remote column. Remove one
named and one unnamed slot.
Expected: A new "Speaker N" row appears (numbered per side, independent of the other column); editor
goes dirty; Save persists it and reopening shows it. Removing a slot renumbers remaining unnamed slots
sequentially. Roster-pick / free-text add in one column never affects the other column.
Record:

## P8. Diarise gate hint while dirty
Steps: Edit any field to dirty the editor, look at the Speakers card; then Save (or Discard).
Expected: Muted line "Save changes before splitting speakers." appears and "Split speakers..." is
disabled while dirty. After Save or Discard the hint clears and the button re-enables.
Record:

## P9. Re-diarise preserves named speaker ownership (EVIDENTIARY)
Steps: On a session where a named speaker is already attached to a detected voice (prior Split
speakers run), run Split speakers again so clustering reassigns/collides ids.
Expected: The named speaker's name stays attached to the SAME voice/lines after re-diarise - never
silently reattached to a different voice; only genuinely new unowned voices get fresh ids.
Record:

## P10. Matter picker - search, bounded list, archived reveal
Steps: In Session Details' Matters area (20+ matters, some archived): type a fragment of a name, then a
reference, then part of an id; then clear the search.
Expected: Results list is capped (~160px, scrollable), never grows the window; each search live-narrows
case-insensitively; clearing restores the full active list. An archived matter appears only when
searched, suffixed " (archived)"; hidden again on empty search. No "Show archived" checkbox in this card.
Record:

## P11. Matter picker - chips, buffered tagging, inline create
Steps: Tick a result (chip appears), remove the chip via its x. Tag a matter, then type a search
matching nothing. Type a name matching no matter and click the create affordance.
Expected: Ticking adds a removable chip and enables Save (dirty); Matters page shows the OLD session
count until Save; Save commits, Discard reverts chips to last-saved. A tagged chip stays visible even
when the search text does not match it, and Save still persists it. Unmatched text shows
`Create matter "<text>"`; clicking it adds the chip AND immediately lists the new matter on the Matters
page with 0 sessions; after Save it shows 1.
Record:

## P12. Sessions grid matter filter (searchable, selection preserved)
Steps: Type a name/reference/id fragment into the Sessions-page matter filter combo. Select a matter as
the filter, then type a search matching nothing.
Expected: Options narrow live; "All matters" and "No matter" sentinels always present. A selected
filter is never dropped/reset by an unrelated search - the grid stays filtered to it.
Record:

## P13. Matters manager search box
Steps: Type into the Matters-page search box; toggle "Show archived" on/off while a search is active.
Expected: List narrows live by name/reference/id. An archived matter's name does NOT reveal it unless
"Show archived" is also checked - search composes with, never overrides, the archived gate.
Record:

## P14. Untag - confirm, open-window refusal, grid refresh
Steps: Tag a session, close its details window. On the Matters page click Untag on the tagged-session
row: choose No, then Yes. Re-tag, open the session's details window, click Untag from Matters. Close
the window, Untag again with Yes.
Expected: Confirm reads `Untag this session from "<Matter Name>"? The session itself is kept; only the
matter tag is removed.` No changes nothing. Yes removes the row + updates the count immediately. While
the session's window is open, Untag shows an Info box `This session is open in another window (Session
Details or read view). Close it first, then untag.` and changes nothing. transcript.jsonl and audio
untouched throughout (only meta.json / matters index change).
Record:

## P15. Both themes readable
Steps: With Session Details open (dirty indicator, Speakers card, chips, Save/Discard) and the Matters
page visible, toggle Windows light/dark theme.
Expected: All text legible in both themes incl. muted "Unsaved changes", diarise hint, chip text;
primary buttons show light text on accent fill; nothing renders invisible.
Record:

# Part B - Phase 3: recording lifecycle & discoverability

## H1. Nav-rail Record (LIVE runtime-verify - reviewers flagged this as the one open framework risk)
Steps: Observe the 48px compact rail; expand the pane. Click Record while Idle. Watch the selection
indicator during and after the click. Click Record again while recording.
Expected: Rail shows a Record icon above Sessions; expanded pane shows the "Record" label. Click starts
a recording one-click (no pre-flight modal). The item greys out while Recording/Paused/Finalizing. The
selection indicator does NOT park on Record; if it visibly flickers, record it - the documented
fallback is the Click-handler variant in the Phase 3 plan's D2 task.
Record:

## H2. Record console (idle)
Steps: Tray menu > "Open record console" while Idle.
Expected: Window titled "LocalScribe - Record console"; centered idle panel with "Ready to record",
Remote/Microphone summary lines matching current Settings, and - only when Remote capture = perProcess -
the "Record this app" selector seeded from Settings > Recording > Per-app target, with the note
"Applies to this recording only. The saved default is unchanged (Settings > Recording)."
Record:

## H3. Console Start
Steps: Click "Start recording" in the console.
Expected: Session starts; the window flips to the live transcript + Pause/Stop controls; the overlay
pill appears; the tray icon turns red.
Record:

## H4. Open-on-Start + Resume does not steal focus
Steps: Close the console. Start a recording from the tray. Then Pause and Resume while another window
has focus.
Expected: Starting opens/activates the console automatically. Pause then Resume does NOT re-activate
or focus-steal the console.
Record:

## H5. Per-session app override (never persists)
Steps: Settings Per-app target = e.g. "CiscoCollabHost". In the console selector type a different image
(e.g. "Zoom"). Start, then Stop. Check session.json and Settings. Reopen the console.
Expected: session.json `devices.remote.app` reflects the override ("Zoom"); Settings > Per-app target
is UNCHANGED; the console selector shows the saved default again after Stop.
Record:

## H6. Settings per-app picker
Steps: Toggle Remote capture between modes; open the Per-app target dropdown; type free text and tab
away; restart the app.
Expected: The row greys out unless Remote capture = perProcess; suggestions CiscoCollabHost / Webex /
Zoom drop down; free text commits on focus loss and survives the restart.
Record:

## H7. Shell status strip
Steps: Observe the bottom strip while Idle, Recording, and Paused; in both themes.
Expected: Idle = grey dot + "Idle", no elapsed. Recording = red dot + "Recording" + ticking elapsed.
Paused = orange dot + "Paused" with elapsed STILL ticking (wall-clock semantics, spec 2.1). Readable in
light AND dark.
Record:

## H8. Sessions empty state
Steps: Launch against a fresh storage root.
Expected: The Sessions grid reads "No sessions yet - press Record to start."
Record:

## H9. Pad-to-end (the Phase-1 deferred audio bug, now fixed write-side)
Steps: Record a short real session, pause a few seconds mid-way, stop. Play it back in the read view;
seek near the very end; double-click a transcript line near the tail.
Expected: The seek bar's total duration matches the session duration (no ~1.7s shortfall); seeking near
the end plays silence instead of clamping early; the tail transcript-jump lands correctly. (The Phase 1
3-layer playback clamp remains as defense.)
Record:

## H10. Consent surfaces unchanged
Steps: Observe tray icon/tooltip/balloons through a record/pause/stop cycle; try Exit during a recording.
Expected: All behave exactly as before this stage; Exit still confirms during a recording and awaits
the stop.
Record:
