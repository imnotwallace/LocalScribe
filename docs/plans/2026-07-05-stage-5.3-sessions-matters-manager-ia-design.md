# Stage 5.3 Design — Sessions & Matters Manager IA

**Goal:** Deliver the "Manager IA overhaul" that Stage 5.2 explicitly deferred (Stage 5.2 design §"Deferred", lines 248-249). Driven by hands-on GUI feedback on the running app: make the Sessions page's row actions discoverable and honest, surface a session's linked matters and its true audio-isolation mode, turn the nav into a space-saving compact rail, and widen the matter-id capacity.

**Context / provenance:** Feedback captured while driving the Stage-5.2 build in the live app (2026-07-05). Six items plus two follow-ups (App-column honesty, matter-id capacity) were brainstormed into the decisions below. Stage 5.2 shipped the Session Details window + two-column speaker manager + cluster picker; this stage does NOT re-open those windows' internals except to add one button (Split speakers) to the Session Details Speakers section.

**Tech stack:** .NET `net10.0-windows`, WPF, Wpf.Ui 4.0.3 (`xmlns:ui="http://schemas.lepo.co/wpfui/2022/xaml"`), CommunityToolkit.Mvvm 8.4.0 (`[ObservableProperty]`, `IRelayCommand`/`IAsyncRelayCommand`), xUnit 2.9.3. Solution `LocalScribe.slnx`.

## Global constraints

- **Evidentiary (memory `no-transcript-content-deletion-evidentiary`):** never mutate/delete transcript data. The Source-column change (§B) is a display change over existing fields; no capture/audio behavior changes.
- **Forward-only matter-id change:** existing matter ids are stable folder names + references (in `meta.matterIds`, the matters index, session tags). The new id format applies to NEWLY-created matters only; old and new coexist. No migration, no rename of existing ids.
- **Zero new build warnings** (`dotnet build LocalScribe.slnx` stays 0/0). App.Tests stay green (the 2 `LocalScribe.Core.Tests` DER/golden fixture failures are pre-existing env-absence, not regressions).
- **Pump-safety / windows** unchanged: no new startup windows; SplitSpeakersWindow already opens on demand.
- **ASCII-only test scripts; no Unicode emojis in tests.** Production glyphs stay `\uXXXX` / XML entities.
- **Commit trailer:** end every commit with `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

## A. Navigation → compact icon rail

**Change:** `MainWindow.xaml` `RootNav` defaults to the 48px icons-only compact rail rather than the 200px labelled pane; the hamburger toggle expands to labels and collapses back. Sessions / Matters / Settings icons are always visible with the active one highlighted.

**Implementation:** set `PaneDisplayMode="LeftCompact"` (compact rail by default, expands on toggle). Keep `CompactPaneLength="48"`, `OpenPaneLength="200"`, `IsPaneToggleVisible="True"`. Pure XAML; no VM/logic change. Confirm at build/smoke that `LeftCompact` in Wpf.Ui 4.0.3 gives the always-visible thin rail + expand-on-hamburger behavior; if the exact enum/behavior differs, use the nearest mode that yields "48px icons default, hamburger expands to labels" (fallback: keep `LeftFluent` but set the pane initial state collapsed).

**Rationale:** only 3 destinations; a thin always-visible icon rail keeps navigation discoverable while returning ~150px of width to the content. Overlay-drawer (LeftMinimal) was considered and rejected (hides nav until toggled — less discoverable for a 3-item nav).

## B. Sessions grid columns

### B1. App → **Source** (fold capture mode inline)
Replace the bare `App` column with a **Source** column that carries the audio-isolation fact inline: `Webex — system mix`, `Webex — per-app`, `Teams — system mix`.

- `SessionRowViewModel` gains `public string Source { get; }` built from `session.App` + the remote mode: `IsSystemMix` (already computed) → suffix `" — system mix"`, else `" — per-app"`. Preserve the existing `App / Medium` disambiguation (when `meta.Medium` differs from `App`, keep showing both, then the mode suffix). Keep the existing `SystemMixTooltip` as the Source cell's tooltip.
- The redundant **"System mix" chip is removed** from the Status column (Source now states it). Status keeps Recovered / Edited / Diarised / Archived chips. `IsSystemMix`/`SystemMixTooltip` stay on the row VM (now consumed by Source instead of the badge).

**Evidentiary rationale:** `App` is a DECLARED target label, not a content guarantee; in system-mix mode the remote leg is the whole system output ("other app audio may be included"). Folding the mode into the source label makes the honesty visible at a glance instead of relegated to a side badge.

### B2. New **Matters** column (chips)
A new column rendering one chip per tagged matter.

- Chip text = **`[ref] [name]`** (e.g. `REF1 Test 1`); when a matter has no `Reference`, chip = name only. Chips wrap within the cell; on overflow show the first few + a `+N` chip. Cell tooltip (and the `+N` chip's tooltip) lists the full `M-20260705-001-REF1 Test 1` for each. `(none)` (muted) when the session has no matter tags.
- Data: `SessionRowViewModel` exposes the resolved chip models for its `MatterIds`. Because the row VM is constructed from a `SessionListItem` (no matter catalog), the **`SessionsPageViewModel` owns the `id → (Reference, Name)` lookup** (see §E "shared enabler") and projects it onto rows — either by passing the lookup into row construction or by exposing a row-level `IReadOnlyList<MatterChip>` the VM fills after the catalog loads. Unknown ids (matter deleted but tag lingering) fall back to showing the raw id.

## C. Row actions → action bar + fixed right-click menu

### C1. Action bar above the grid
A fixed toolbar row directly above the `SessionsGrid`, left-aligned: **View transcript** · **Session details** · **Reveal in Explorer** · **Archive/Unarchive** · **Delete**.

- Buttons are **enabled only when a row is selected** (bind `IsEnabled` to `SelectedRow != null`, e.g. via the command's `CanExecute` or a converter). Delete keeps its existing confirmation flow. The Archive button label reflects the selected row's archived state (Archive vs Unarchive).
- Buttons bind the EXISTING commands (no new command logic): View transcript → `OpenReadViewCommand`, Session details → `OpenSessionDetailsCommand`, Reveal → `RevealInExplorerCommand`, Archive → `ToggleArchiveCommand`, Delete → `DeleteSessionCommand`, all with `CommandParameter="{Binding SelectedRow}"`.

### C2. Right-click menu retained + bug #3 fixed
Keep a right-click context menu with the SAME five actions. **Fix the bug** where right-clicking over the Title/text cells surfaces WPF's default `Copy/Cut/Paste` text menu instead of the row menu: attach the `ContextMenu` at the DataGrid **row** level (`DataGridRow` style) and/or suppress the text-edit context menu on the read-only text cells, so the row menu wins on every column. (The DataGrid is already `IsReadOnly="True"`; the fix ensures the cell TextBlocks don't expose an editing menu.)

### C3. Renames + removal
- `Open` → **View transcript**; `Open detail` → **Session details** (commands/behavior unchanged, labels only). Double-click a row still triggers View transcript (`OpenReadViewCommand`).
- **Split speakers removed** from both the action bar and the context menu (relocated per §D).

## D. Split speakers relocation

Split speakers stops being a Sessions-list action and becomes an in-session action from both in-session surfaces:

- **Read view toolbar:** unchanged — it already has its "Split speakers..." button wired to `openSplitSpeakers`.
- **Session Details window (Speakers section):** ADD a "Split speakers..." button. Wiring mirrors the existing read-view path: `MetadataEditorViewModel` gains a `DiariseCommand` (`IRelayCommand`) + `event Action<string>? DiariseRequested` that raises the session id; the App `openSessionDetails` factory wires `detailEditor.DiariseRequested += openSplitSpeakers`. The button's enablement reuses the SAME diarisable gating as today (finalized, not pending-recovery, has a splittable remote leg — reuse the predicate the existing `DiariseCommand`/read view uses; do not invent new gating).
- **SessionsPageViewModel cleanup:** with the Sessions-list entry point gone, `DiariseCommand` and `DiariseRequested` on `SessionsPageViewModel` become dead. Remove them (and the `App.xaml.cs` wiring `sessionsVm.DiariseRequested += openSplitSpeakers`), after grep-confirming no other reference — mirroring the Stage-5.2 `JumpToSessionRequested` cleanup. `openSplitSpeakers` itself stays (read view + Session details use it).

## E. Matter identity

### E1. Matter-id format `M-YYYYMMDD-NNN` (per-day)
`MatterIdGenerator.Next` mints `M-{yyyyMMdd}-{NNN:D3}` — sequence resets per DAY (999/day). Change the parameter from `int year` to the creation date (`DateTimeOffset` or `DateOnly`); the prefix becomes `M-{date:yyyyMMdd}-`; the max-scan and free-id loop are otherwise unchanged (scan entries sharing the DAY prefix, increment `NNN:D3`, ensure both the index id and `matters/<id>/` folder are free). Update the caller (`MaintenanceService.CreateMatterAsync`) to pass the creation date. Keep invariant culture and the orphan-folder-safety loop verbatim.

**Coexistence:** existing `M-2026-0NN` ids are never touched or reissued; the per-day scan simply won't collide with year-format ids (different prefix). Both display fine through §E2/§B2 (any id string).

### E2. Matter filter dropdown label `[id]-[ref] [name]`
`RebuildMatterOptions` builds each `MatterFilterOption.Label` as **`{id}-{ref} {name}`** (e.g. `M-20260705-001-REF1 Test 1`; omit `-{ref}` when a matter has no reference; the value stays the matter id). "All matters" (`Id=null`) and "No matter" (`NoMatterSentinel`) specials unchanged. Options are still derived from the matter ids present on rows, but labels now use the matter catalog's `(Reference, Name)`.

### E3. Shared enabler — matter catalog lookup in `SessionsPageViewModel`
Both §B2 (chips) and §E2 (filter labels) need each matter's `(Reference, Name)`. The Sessions VM loads the matters index — `MatterStore(paths.MattersDir).ListAsync(ct)` → `MattersIndex.Matters` (each `MattersIndexEntry` has `Id`/`Name`/`Reference`) — into an `id → (Reference, Name)` dictionary, refreshed on navigation-in alongside the existing session refresh (`OnNavigatedToAsync`). Reuse whatever matter-listing seam already exists (the `MattersPageViewModel` lists matters via the maintenance layer; add a thin `MaintenanceService.ListMattersAsync`-style accessor if none is directly reusable). Chips/filter resolve through this dict; a missing id degrades to the raw id.
