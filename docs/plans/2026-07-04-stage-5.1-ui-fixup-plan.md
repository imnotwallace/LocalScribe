# Stage 5.1 UI Fixup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make LocalScribe look and behave like a finished native Windows tray app — one consistent Fluent theme that follows the OS, a branded icon, working window chrome, no clipped controls, and a readable, grouped Sessions detail editor.

**Architecture:** Pure UI/UX remediation over the existing WPF-UI (Wpf.Ui 4.0.3) shell. No changes to capture, transcription, diarisation, persistence, or schema, and no ViewModel API changes (the humble-object split stays intact — every VM command/property keeps its current name and behaviour). Work is XAML + a small amount of startup/host code-behind, backed by an app-level shared resource dictionary that becomes the single design-language source.

**Tech Stack:** .NET `net10.0-windows`, WPF, Wpf.Ui 4.0.3 (`xmlns:ui="http://schemas.lepo.co/wpfui/2022/xaml"`), H.NotifyIcon.Wpf 2.3.0, CommunityToolkit.Mvvm 8.4.0, xUnit 2.9.3.

## Global Constraints

- **No ViewModel API changes.** VMs are WPF-free and already tested (449 tests). Tasks may relabel UI text and regroup controls but MUST preserve every binding path, command name, and property name. If a rename is truly unavoidable, update its test in the same task.
- **Zero new build warnings.** The project ships on a 0-warning gate; `dotnet build` MUST stay warning-clean.
- **Full suite green.** `dotnet test` (LocalScribe.App.Tests + LocalScribe.Core.Tests) MUST stay green after every task.
- **Do not edit LocalScribe.Core or its tests.** This stage is App-only.
- **ASCII-only in test scripts; no Unicode emojis in tests** (user rule). Glyph literals in production code stay as `\uXXXX` escapes (existing convention, TrayIconHost.cs:156).
- **Pump-safety (memory `wpfui-fluentwindow-startup-rendering`):** never show a Mica `FluentWindow` synchronously before the message pump. Keep the `ApplicationIdle` deferral for `OpenMainWindow` (App.xaml.cs:184). Any new theme/tray work at startup that renders off-screen must run at/after `ApplicationIdle`.
- **Evidentiary (memory `no-transcript-content-deletion-evidentiary`):** WS7 only makes accidental session-folder deletion harder; it adds no content-removal capability.
- **Verified Wpf.Ui 4.0.3 API (reflected from the installed assembly — use exactly these):**
  - `Wpf.Ui.Appearance.ApplicationThemeManager.ApplySystemTheme()` and `.Apply(ApplicationTheme, WindowBackdropType backgroundEffect = Mica, bool updateAccent = true)`; `.GetAppTheme()`.
  - `Wpf.Ui.Appearance.SystemThemeWatcher.Watch(Window window, WindowBackdropType backdrop = Mica, bool updateAccents = true)` and `.UnWatch(Window)`.
  - `ApplicationTheme` enum: `Unknown | Dark | Light | HighContrast`.
  - Controls: `ui:TitleBar` (props `Title`, `Header`, `Icon`, `ShowMinimize/ShowMaximize/ShowClose`, `MinimizeActionOverride`, `MaximizeActionOverride`, `CanMaximize`), `ui:DataGrid`, `ui:Card`, `ui:CardExpander`, `ui:TextBlock`, `ui:Button`.
  - `ui:Button` has `Appearance` of type `Wpf.Ui.Controls.ControlAppearance` enum: `Primary | Secondary | Info | Dark | Light | Danger | Success | Caution | Transparent`.
- **Shared Fluent theme brush keys used by this plan** (standard Wpf.Ui 4.x resource keys, applied via `{DynamicResource ...}`): `TextFillColorPrimaryBrush`, `TextFillColorSecondaryBrush`, `ControlFillColorSecondaryBrush`, `SystemFillColorCautionBrush`, `AccentFillColorDefaultBrush`. Correctness of these keys is validated by manual smoke E1/E2 (a wrong key renders as a silent fallback); the static test only guards against regressing to hardcoded literals.

## Shared design vocabulary (created in Task 1, referenced everywhere after)

New file `src/LocalScribe.App/Styles/Fluent.Shared.xaml` (a `ResourceDictionary`) is merged app-wide in `App.xaml`. It declares exactly these keys — every later task references them by these names:

| Key | Type | Purpose |
|---|---|---|
| (implicit) | `Style TargetType="TextBlock"` | `Foreground = {DynamicResource TextFillColorPrimaryBrush}` — fixes black-on-dark for all unstyled text |
| `MutedText` | `Style TargetType="TextBlock"` | `Foreground = {DynamicResource TextFillColorSecondaryBrush}` — replaces `Foreground="Gray"` / `Opacity` hints |

| `FieldLabel` | `Style TargetType="TextBlock"` | lifted verbatim from SettingsPage (VerticalAlignment Center, Margin 0,0,8,0, MinWidth 140) |
| `FieldRow` | `Style TargetType="StackPanel"` | lifted verbatim (Orientation Horizontal, Margin 0,4) |
| `Note` | `Style TargetType="TextBlock"` | lifted verbatim (TextWrapping Wrap, Opacity 0.7, Margin 0,4,0,0) |
| `SectionCard` | `Style TargetType="{x:Type ui:Card}"` | Margin 0,0,0,12, Padding 12 — the grouped-section container |

Buttons use `ui:Button Appearance="Primary|Secondary|Danger"` directly (no custom style key).

Create `tools/generate-icon/Program.cs`. It renders a simple LocalScribe mark — a microphone glyph inside a rounded accent square — at multiple sizes and writes a valid multi-frame `.ico` (PNG-compressed frames, valid on Windows Vista+). The glyph is drawn with GDI+ primitives (no external font dependency, so it is deterministic on any build box):

```csharp
using System.Drawing;
using System.Drawing.Drawing2D;
using System.Drawing.Imaging;

// LocalScribe mark: rounded accent-blue tile + white microphone + baseline (transcript) strokes.
static Bitmap Render(int s)
{
    var bmp = new Bitmap(s, s, PixelFormat.Format32bppArgb);
    using var g = Graphics.FromImage(bmp);
    g.SmoothingMode = SmoothingMode.AntiAlias;
    g.Clear(Color.Transparent);

    float pad = s * 0.06f, r = s * 0.22f;
    var tile = new RectangleF(pad, pad, s - 2 * pad, s - 2 * pad);
    using (var path = RoundedRect(tile, r))
    using (var fill = new LinearGradientBrush(tile, Color.FromArgb(0x2B, 0x88, 0xD8),
               Color.FromArgb(0x1A, 0x5F, 0xB4), LinearGradientMode.ForwardDiagonal))
        g.FillPath(fill, path);

    using var white = new SolidBrush(Color.White);
    // Mic capsule.
    float mw = s * 0.20f, mh = s * 0.34f, cx = s * 0.5f, top = s * 0.24f;
    g.FillPath(white, RoundedRect(new RectangleF(cx - mw / 2, top, mw, mh), mw / 2));
    // Mic arc + stand.
    using var pen = new Pen(white, MathF.Max(1f, s * 0.045f)) { StartCap = LineCap.Round, EndCap = LineCap.Round };
    float ar = s * 0.16f, ay = top + mh - ar;
    g.DrawArc(pen, cx - ar, ay, 2 * ar, 2 * ar, 20, 140);
    g.DrawLine(pen, cx, ay + ar, cx, ay + ar + s * 0.08f);
    g.DrawLine(pen, cx - s * 0.10f, ay + ar + s * 0.08f, cx + s * 0.10f, ay + ar + s * 0.08f);
    return bmp;
}

static GraphicsPath RoundedRect(RectangleF b, float r)
{
    var p = new GraphicsPath();
    p.AddArc(b.X, b.Y, 2 * r, 2 * r, 180, 90);
    p.AddArc(b.Right - 2 * r, b.Y, 2 * r, 2 * r, 270, 90);
    p.AddArc(b.Right - 2 * r, b.Bottom - 2 * r, 2 * r, 2 * r, 0, 90);
    p.AddArc(b.X, b.Bottom - 2 * r, 2 * r, 2 * r, 90, 90);
    p.CloseFigure();
    return p;
}

int[] sizes = { 16, 32, 48, 64, 128, 256 };
var frames = sizes.Select(Render).ToArray();
var pngs = frames.Select(f => { using var ms = new MemoryStream(); f.Save(ms, ImageFormat.Png); return ms.ToArray(); }).ToArray();

string outPath = args.Length > 0 ? args[0] : "LocalScribe.ico";
using var fs = new FileStream(outPath, FileMode.Create);
using var w = new BinaryWriter(fs);
w.Write((short)0); w.Write((short)1); w.Write((short)sizes.Length);   // ICONDIR
int offset = 6 + 16 * sizes.Length;
for (int i = 0; i < sizes.Length; i++)                                 // ICONDIRENTRY[]
{
    w.Write((byte)(sizes[i] >= 256 ? 0 : sizes[i]));                   // width  (0 == 256)
    w.Write((byte)(sizes[i] >= 256 ? 0 : sizes[i]));                   // height
    w.Write((byte)0); w.Write((byte)0);                               // colors, reserved
    w.Write((short)1); w.Write((short)32);                            // planes, bpp
    w.Write(pngs[i].Length); w.Write(offset);
    offset += pngs[i].Length;
}
foreach (var png in pngs) w.Write(png);
Console.WriteLine($"Wrote {outPath} ({sizes.Length} frames)");
```

    var offenders = new List<string>();
    foreach (var file in Directory.EnumerateFiles(appDir, "*.xaml", SearchOption.AllDirectories))
    {
        foreach (var line in File.ReadLines(file))
        {
            var m = System.Text.RegularExpressions.Regex.Matches(line, "#[0-9A-Fa-f]{6,8}\\b");
