# Stage 5.3 smoke runbook - Sessions & Matters Manager IA (compact nav, action bar, matters chips, per-day ids)

GUI-only acceptance (cannot be run headless). Run on the Win11 box against a real build and record
results inline at the bottom. Fix any failure before finishing the branch.

Run: `dotnet run --project src/LocalScribe.App`

## Prerequisites

- Build is green: `dotnet build LocalScribe.slnx` (0 warnings / 0 errors); `dotnet test tests/LocalScribe.App.Tests`
  (181/181). The 2 `LocalScribe.Core.Tests` fixture failures (`Der_within_baseline_plus_epsilon`,
  `Golden_pair_wer_stays_at_baseline`) are pre-existing env-absent DER/golden fixtures - not Stage 5.3
  regressions.
- At least one **finalized** session (for G2/G3/G7). A second finalized session helps G3's "acts on the
  right-clicked row, not the selected one" check.
- At least one **pending / in-progress** (not yet finalized) session, for G7's disabled-button check.
- At least two **Matters**: one with a reference set and one WITHOUT a reference, each tagged to a
  session, plus one **untagged** session - so G5/G6 can see both chip/label formats and the blank case.
- If one exists from before this stage, an existing **legacy matter** (id format `M-YYYY-NNN`) for the
  forward-only display check in G8. If none exists, note G8's legacy half as N/A rather than failing it.
- For G7's Split-speakers dialog to actually RUN end-to-end (not just open/disabled-check), the diariser
  prerequisite from the Stage 5 runbook applies: `LocalScribe.Diarizer.exe` must be published beside the
  app (self-contained single-file with `-p:IncludeNativeLibrariesForSelfExtract=true`) or Run fails with
  `HELPER_CRASH`. See `docs/plans/2026-07-04-stage-5-smoke-runbook.md` "Prerequisite" for the exact
  publish + copy-only-the-.exe steps. Confirming the button opens the dialog and the disabled state is
  correct does NOT require this - only running a split to completion does.

## G1 - Nav starts as a compact icon rail; hamburger expands/collapses it

Steps: launch the app fresh and observe the left nav on load. Click the hamburger (pane toggle). Click it
again.
Expected: on launch the left nav is a thin 48px icon-only rail (Sessions / Matters / Settings icons only,
no labels; the active item is highlighted). Clicking the hamburger expands it to the 200px labelled pane
(icons + text); clicking again collapses it back to the icon rail. The content area (Sessions/Matters/
Settings pages) is visibly wider than the old always-open 200px pane.
Record: pass/fail. IMPLEMENTATION NOTE: this uses `PaneDisplayMode="LeftFluent"` + `IsPaneOpen="False"`
(a fallback - `LeftCompact` does not exist in Wpf.Ui 4.0.3). If the nav shows NO rail (just a bare
hamburger with no icons), record FAIL with a note - the known fix is `PaneDisplayMode="Left"`.

## G2 - Action bar above the Sessions grid

Steps: on the Sessions page, with NO row selected, observe the button bar above the grid (View transcript
/ Session details / Reveal in Explorer / Archive / Unarchive / Delete...). Select a row. Exercise each
button in turn.
Expected: with no selection all buttons are DISABLED (greyed). Selecting a row enables them. View
transcript opens the read-view window for that session; Session details opens the Session Details window
for that session; Reveal in Explorer opens the session's folder in Windows Explorer; Archive toggles the
row's Archived state (badge appears) and Unarchive toggles it back; Delete... shows the confirm dialog
(delete to Recycle Bin).
Record: pass/fail + confirm the no-selection disabled state + confirm each button's action.

## G3 - Right-click bug fixed (Title column) + acts on the right-clicked row

Steps: right-click directly over the **Title** column's cell text for a session row that is NOT the
currently selected row (select a different row first, if any is selected). Choose an item from the menu
(e.g. Session details) and confirm which session it opened.
Expected: the context menu over the Title cell now shows the SESSION menu: View transcript / Session
details / Reveal in Explorer / Archive / Unarchive / (separator) / Delete (to Recycle Bin)... - NOT the
old text-cell Copy/Cut/Paste menu. The chosen command acts on the ROW YOU RIGHT-CLICKED, not on whatever
row was selected beforehand. No "Split speakers" item appears in this menu or in the action bar (G2).
Record: pass/fail + confirm the menu is the session menu (not Copy/Cut/Paste) + confirm the command
targeted the right-clicked row + confirm no Split speakers item anywhere in Sessions.

## G4 - Source column replaces App; System mix badge dropped

Steps: on the Sessions grid, read the "Source" column (previously "App") for a system-mix session, a
per-app session, and, if available, a session with a differing local/remote medium. Also check the
status-badge area for a system-mix session.
Expected: Source reads e.g. "Webex - system mix" or "Webex - per-app" (an em dash separates the app name
and capture mode); a differing-medium session reads e.g. "Webex / Teams - system mix". The separate
"System mix" status badge that used to appear alongside the other chips is GONE; the other status chips
(Recovering / Recovered / Edited / Diarised / Archived) still appear as before.
Record: pass/fail + example Source text seen + confirm the standalone "System mix" badge no longer
appears + confirm the other badges are unaffected.

## G5 - Matters chips column

Steps: look at the Matters column for: a session tagged to a matter WITH a reference, a session tagged to
a matter with NO reference, and an untagged session. Hover the chip on each tagged row.
Expected: a session tagged to a matter with a reference shows a chip reading "[ref] name"; a matter with
no reference shows just "name". Hovering a chip shows the full tooltip "M-YYYYMMDD-NNN-REF name" (or
"M-YYYYMMDD-NNN name" without a ref). An untagged session shows a blank Matters cell. (If constructible, a
stored matter id with no matching catalog entry shows the raw id as a fallback, no crash.)
Record: pass/fail + confirm chip text for both ref/no-ref cases + confirm the full-id tooltip text +
confirm blank for the untagged session.

## G6 - Matter filter dropdown labels

Steps: open the Sessions toolbar's matter-filter dropdown and read through its options. Select a matter
option (not "All matters"/"No matter").
Expected: each matter option reads "M-YYYYMMDD-NNN-REF name" (or "M-YYYYMMDD-NNN name" when the matter has
no reference). "All matters" and "No matter" options remain present. Selecting a matter option filters the
Sessions grid down to that matter's tagged sessions.
Record: pass/fail + example option text seen + confirm "All matters"/"No matter" still present + confirm
filtering works.

## G7 - Split speakers relocated into Session Details; disabled while pending

Steps: open the Session Details window for a FINALIZED session and find "Split speakers..." in the
Speakers section; click it. Separately, open the Session Details window for a PENDING / in-progress (not
yet finalized) session and check the same button's state. Then confirm the read-view toolbar's own
Split-speakers control still works, unchanged.
Expected: for a finalized session, "Split speakers..." opens the diarisation (Split speakers) dialog for
that session (same dialog as before Stage 5.3, launched from a new place). For a pending/in-progress
session the button is DISABLED (greyed, not clickable). The read-view toolbar's Split-speakers button is
unaffected and still works. (If `LocalScribe.Diarizer.exe` isn't published beside the app per the
Prerequisites note, the dialog may fail with `HELPER_CRASH` once run - that's an environment issue, not a
G7 failure; record it separately from whether the button opened the dialog and the disabled state was
correct.)
Record: pass/fail + confirm finalized -> dialog opens + confirm pending -> button disabled + confirm the
read-view Split-speakers control is unaffected.

## G8 - Per-day matter id

Steps: on the Matters page, create a NEW matter today; note its id. Create a second new matter the same
day; note its id. If an existing legacy matter (id format "M-YYYY-NNN") is present from before this stage,
look at it too.
Expected: each new matter's id is "M-YYYYMMDD-NNN" using TODAY's (UTC) date, e.g. M-20260705-001, then
M-20260705-002 for the second one created the same day (increments per day). It displays correctly in the
matter filter dropdown (G6) and in row chips (G5). Any existing legacy "M-YYYY-NNN" matter still displays
fine everywhere (forward-only change; legacy ids are never rewritten).
Record: pass/fail + the actual ids minted + confirm they display correctly in G6/G5 + confirm a legacy
matter (if any) still displays fine.
